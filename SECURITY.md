# Security policy

Please do not report security issues in public issues or pull requests.

## How to report

Send a private report to the maintainers including:
- description and impact
- affected versions (if known)
- reproduction steps or proof-of-concept (if safe)
- suggested fix (optional)

## What counts as a security issue here

Examples:
- integrity verification bypass
- provenance/identity spoofing issues
- supply-chain risks introduced by dependencies or CI
- vulnerability in parsing/validation that could cause compromise in downstream systems

## Response

Maintainers will:
- acknowledge receipt
- assess severity and scope
- coordinate a fix and release
- publish a short advisory if warranted
